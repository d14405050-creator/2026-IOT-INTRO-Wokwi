# Week 13 Homework：SH1107 + DHT22 中文動畫溫度看板

## 作業主題
請以 `in-class/task3a` 與 `in-class/task4` 為基礎，完成一個 **SH1107 OLED 中文動畫溫度看板**。

本作業重點是把：
- `task3a` 的畫面設計與動畫能力
- `task4` 的 DHT22 感測整合

整合成一個可持續運作、可觀察、可驗收的完整作品。

## 參考版型（必須對齊）
版型需明確對齊下列參考視覺風格（不要求像素級完全一致，但需看得出同一版型邏輯）：

![img](in-class/imgs/img11.gif)

版型元素建議包含：
- 龍珠主視覺（圓 + 星）
- 英文資訊文字區塊
- 中文字動態區塊（如「花火節」）
- 新增溫度資訊區塊（`xx.x C`）

## 必做需求（100% 必須完成）
1. **Layout 要符合參考圖風格**
- 需有清楚的主視覺層次（標題/資訊/圖形/動畫）
- 版面不可只有單行溫度文字

2. **必須有動畫**
- 畫面需持續更新（例如每 200~300ms 更新一幀）
- 可使用放大縮小、波浪位移、閃爍或循環切換

3. **必須呈現中文字**
- 至少 3 個中文字（建議沿用「花火節」）
- 中文需可辨識，不可破圖或錯位到完全不可讀

4. **必須有即時溫度顯示**
- 溫度來源必須是 DHT22 感測值
- 顯示格式：`xx.x C`
- 建議每 2 秒更新一次感測值

5. **OLED（LED）畫面需加入學號識別**
- 畫面中必須可看見學號（可縮寫或完整學號）
- 學號不可只出現在 README，必須實際顯示在 OLED 畫面

## 感測與除錯要求
1. 每次讀值後，Serial Monitor 印出 debug 訊息，至少包含：
- `temperature`
- `humidity`

2. 若讀值失敗（例：暫時性讀取錯誤），程式不可直接當掉：
- 要有 `try/except`
- Serial 顯示錯誤訊息
- OLED 顯示錯誤狀態（例如 `Sensor Error`）或保留前次有效值

## 技術限制
1. 顯示器需使用 `board-grove-oled-sh1107`（128x128）
2. 請使用 `sh1107` driver，不可改成 `ssd1306`
3. 需保留可調校常數（例如 `center_x`, `center_y`, `offset`）
4. 程式需可連續執行至少 30 秒，不可出現明顯卡死

## 建議實作架構（可調整）
```python
while True:
    # A. 依時間/幀數更新動畫
    # B. 每 2 秒讀一次 DHT22
    # C. 重畫整個畫面（背景 + 中文動畫 + 溫度 + 主圖）
    # D. oled.show()
```

建議函式拆分：
- `draw_static_scene(...)`
- `draw_chinese_animation(...)`
- `draw_temperature(...)`
- `read_sensor_safe(...)`

## 評分標準（100 分）
1. **版型符合參考圖風格（25 分）**
- 25：版型層次清楚，明顯對齊參考風格
- 15：有主要元素，但排版較混亂
- 5：元素缺漏，難以辨識版型

2. **動畫效果（25 分）**
- 25：動畫連續、節奏明顯、可觀察
- 15：有動畫但變化很小或不穩定
- 5：幾乎靜態或僅偶發變化
- 0：未附動畫 GIF（此項不給分）

3. **中文字呈現（25 分）**
- 25：中文字清楚、可辨識、位置合理
- 15：可辨識但有明顯破圖/錯位

4. **溫度顯示與感測整合（25 分）**
- 25：DHT22 讀值穩定、溫度顯示正確、debug 完整
- 15：可讀值但偶發錯誤未妥善處理

5. **學號識別顯示（10 分）**
- 10：OLED 畫面可清楚辨識學號
- 5：有學號但位置/清晰度不佳
